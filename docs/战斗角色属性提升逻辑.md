# 战斗角色属性提升逻辑

本文档记录当前战斗相关的角色属性提升与结算逻辑。当前属性计算主要发生在后端；小程序前端只展示后端返回的 `character` 和 `active_battle` 状态，不参与战斗属性计算。

## 核心链路

1. 玩家选择 `resolution_mode = "combat"` 的事件选项后，后端进入战斗流程。
2. `EventResolutionService._start_combat_event(...)` 调用 `CombatStatService.build_player_state(run)`，基于当前 `RunState` 生成玩家战斗快照。
3. 战斗过程中使用该快照里的气血、攻击、防御、速度结算，不再动态读取角色成长或装备变化。
4. 战斗结束时，`EventResolutionService._finalize_battle(...)` 只把战斗快照里的剩余气血回写到 `run.character.hp_current`。
5. 胜利、失败、逃跑成功分别按事件配置或敌人模板奖励继续结算事件结果。

关键代码：

- `app/core_loop/services/event_resolution_service.py`
  - `_start_combat_event(...)`
  - `_finalize_battle(...)`
- `app/core_loop/services/combat_stat_service.py`
  - `CombatStatService.build_player_state(...)`
- `app/core_loop/services/combat_service.py`
  - `CombatService.perform_action(...)`

## 玩家战斗属性公式

玩家战斗快照字段为 `CombatActorState`：

- `hp_current`
- `hp_max`
- `attack`
- `defense`
- `speed`

计算入口：

```python
CombatStatService.build_player_state(run)
```

### 进度加成

先计算当前境界内修为进度：

```text
local_progress = max(0, character.cultivation_exp - 当前境界入口累计修为)
progress_bonus = min(6, local_progress // 30)
```

`progress_bonus` 有固定上限：

```text
progress_bonus 上限 = 6
```

说明：

- “当前境界入口累计修为”来自当前境界之前所有境界的 `required_exp` 求和。
- `required_exp` 来自境界配置，配置文件是 `config/realms/realms.json`。

### 气血

```text
hp_max = character.hp_max + equipment_bonus.hp_max
hp_current = min(hp_max, character.hp_current + equipment_bonus.hp_max)
```

效果：

- 装备增加气血上限时，进入战斗快照会同步增加当前气血。
- 当前气血不会超过快照气血上限。
- 战斗结束后回写到角色时，会再限制在 `character.hp_max` 以内；因此装备带来的临时上限不会永久写入角色基础气血上限。

### 攻击

```text
attack = 10
  + realm.order_index * 2
  + progress_bonus
  + equipment_bonus.attack
```

### 防御

```text
defense = 3
  + realm.stage_index * 2
  + progress_bonus
  + realm.order_index // 4
  + equipment_bonus.defense
```

### 速度

```text
speed = 6
  + realm.stage_index
  + progress_bonus // 2
  + realm.order_index // 5
  + equipment_bonus.speed
```

## 属性提升来源

当前真正参与战斗数值提升的来源有三类。

### 1. 境界配置

境界配置提供：

- `display_name`
- `stage_index`
- `order_index`
- `required_exp`

其中：

- `order_index` 影响攻击、防御、速度。
- `stage_index` 影响防御、速度。
- `hp_max` 是该境界建议基础气血上限，突破成功时会提升角色基础气血上限。

配置文件：

```text
config/realms/realms.json
```

### 2. 当前境界内修为进度

`character.cultivation_exp` 会通过 `progress_bonus` 转换为战斗属性。当前境界内每 30 点修为提供 1 点进度加成，最多 6 点。

影响：

- 攻击：每 1 点 `progress_bonus` 增加 1 点攻击。
- 防御：每 1 点 `progress_bonus` 增加 1 点防御。
- 速度：每 2 点 `progress_bonus` 增加 1 点速度。

### 3. 已装备装备

装备加成来自 `character.equipped_items` 中当前已装备的装备，并且装备 ID 必须仍存在于 `character.equipment_tags`。

统计字段：

```text
attack
defense
speed
hp_max
```

计算入口：

```python
EquipmentService.get_equipped_stat_bonus(run)
```

装备配置文件：

```text
config/equipment/items.json
```

当前配置示例：

| equipment_id | 槽位 | 当前战斗属性影响 |
| --- | --- | --- |
| `iron_sword` | `weapon` | `attack +4` |
| `cloth_armor` | `armor` | `defense +2`, `hp_max +12` |
| `jade_token` | `accessory` | 无直接战斗属性加成 |
| `bronze_bell` | `artifact` | 无直接战斗属性加成 |

注意：

- 当前配置仓库中的装备 `speed` 固定按 0 读取，配置文件里也没有独立 `speed` 字段。
- `special_effects` 当前不会进入 `CombatStatService` 的属性公式。

## 战斗结算如何使用属性

### 行动先后

每回合判断：

```text
player_first = player.speed >= enemy.speed
```

玩家速度大于或等于敌人速度时，玩家先行动。

### 攻击伤害

伤害公式：

```text
random_multiplier = 0.9 + random() * 0.2
raw_damage = int(attack * random_multiplier) - defense
damage = max(1, raw_damage)
```

即：

- 攻击值先乘以 0.9 到 1.1 的随机倍率。
- 再减去目标防御。
- 最终伤害最低为 1。

### 防御动作

玩家选择 `defend` 时，本回合敌人对玩家造成的伤害降低：

```text
defended_damage = max(1, int(original_damage * 0.6))
```

### 逃跑概率

逃跑成功率：

```text
raw_rate = flee_base_rate + (player.speed - enemy.speed) * 0.03
success_rate = clamp(raw_rate, 0.15, 0.85)
```

速度差每领先 1 点，逃跑成功率增加 3%；每落后 1 点，降低 3%。最终成功率限制在 15% 到 85% 之间。

### 战斗丹药

当前战斗丹药只支持恢复气血：

```text
item.usable_in_battle == true
item.effect_type == "hp_restore"
```

恢复量：

```text
heal_amount = int(item.effect_value * item.effect_multiplier)
```

恢复后不会超过战斗快照的 `player.hp_max`。

## 当前未接入战斗属性提升的内容

以下字段或系统当前没有进入战斗属性提升公式：

- 装备 `special_effects`
- 丹药提供攻击、防御、速度加成
- 状态 `statuses`
- 功法 `techniques`
- 洞府设施加成
- 临时战斗 Buff
- 战斗开始后的装备变化或修为变化

## 维护注意事项

1. 若要新增攻击、防御、速度类 Buff，优先扩展 `CombatStatService.build_player_state(...)`，因为这是战斗快照生成的唯一入口。
2. 若 Buff 需要在战斗中途动态变化，应改动 `ActiveBattleState` 或 `CombatService.perform_action(...)`，不能只改角色基础状态。
3. 若装备 `special_effects` 要生效，需要明确定义其战斗语义，并在属性快照或战斗结算阶段消费。
4. 若希望装备气血上限在战斗结束后仍影响当前气血，需要重新审视 `_finalize_battle(...)` 里回写 `character.hp_current` 时使用 `character.hp_max` 限制的逻辑。
