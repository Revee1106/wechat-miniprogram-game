# 项目记忆

- 事件结果变化中的 `alchemy_mastery_exp_delta` 是炼丹熟练度变动字段，前端必须保留在“新增数值变化”分类中，后端解析、校验和结算链路也不能回退。
- 事件结果变化中的 `learned_alchemy_recipe_ids` 是学习丹方字段，前端应使用丹道配置里的丹方下拉选择，不要改回自由输入。
- 物品变化的材料选项必须来自控制台“材料配置”，不要恢复旧的硬编码资源列表。
- 事件结果变化中的 `unlocked_material_ids` 是解锁材料字段，前端归在“新增附加变化 -> 解锁材料”。洞府随机产出需要先检查该材料是否已解锁，不要让高品级材料绕过事件解锁直接产出。
- 事件选项的 `next_event_id` 是后续事件解锁字段：被任何选项引用为后续事件的模板，在进入 `run.unlocked_event_ids` 前不能进入随机池；玩家实际选择并结算该选项后才写入解锁列表。
