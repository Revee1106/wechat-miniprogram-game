# 洞府系统 V1 进度说明

**日期：** 2026-03-30

本次实现按《问道长生》洞府系统 V1 文档推进，但采用“洞府主循环先完成、炼丹闭环暂缓”的收敛范围。

## 本次已完成

### 1. 洞府运行态

- `RunState` 已支持洞府设施列表
- `RunState` 已支持最近一次洞府月度结算结果
- 保留 `dwelling_level` 作为洞府总览等级，按已建造设施最高等级汇总

### 2. 首版 4 个核心设施

- `灵田`
  - 可建造
  - 可升级到 3 级
  - 月维护扣灵石
  - 月产出基础灵植
- `矿洞`
  - 可建造
  - 可升级到 3 级
  - 月维护扣灵石
  - 月产出基础矿材
- `聚灵阵`
  - 可建造
  - 可升级到 3 级
  - 月维护扣灵石
  - 月产出固定修为
  - 突破时继续提供额外成功率加成
- `炼丹房`
  - 可建造
  - 可升级到 3 级
  - 月维护扣灵石
  - 仅作为功能入口占位
  - 会显示“炼丹入口已解锁，完整玩法后续开放”

### 3. 月度结算

- 推进时间时会先执行洞府月度维护与产出结算
- 若维护费足够：
  - 正常扣费
  - 正常发放产出 / 修为
- 若维护费不足：
  - 设施本月停摆
  - 不发放产出
- 最近一次结算结果会保留在运行态中，供前端洞府页展示

### 4. API 与前端

- 新增 API：
  - `POST /api/run/dwelling/build`
  - `POST /api/run/dwelling/upgrade`
- 小程序 `run-store` 已支持：
  - `buildDwellingFacility`
  - `upgradeDwellingFacility`
- 洞府页已从静态说明页改为可操作页面，包含：
  - 洞府总览
  - 最近一次月结算
  - 设施列表
  - 建造 / 升级按钮
  - 炼丹房占位提示

## 本次未完成

### 1. 炼丹闭环

- 还没有独立炼丹 API
- 还没有丹方配置
- 还没有丹药产物生成
- 炼丹房目前只承担功能占位，不承担资源转化

### 2. 更深层洞府经营

- 没有布局 / 地块系统
- 没有耐久衰减与修复流程
- 没有相邻加成
- 没有弟子 / 仆从 / 排班
- 没有多段生产链
- 没有自动售卖

### 3. 更细资源约束

- 本次洞府建造与升级成本以灵石为主
- 还没有把辅材消耗扩展到完整材料体系
- 洞府月产出目前仍直接服务现有运行态资源字段

## 为后续保留的接口

- `alchemy_room` 已作为设施常驻存在，可直接继续补炼丹入口与操作流
- 洞府设施状态、维护、产出、功能说明已经结构化，可继续加新设施
- 最近一次月结算结构已存在，可继续扩展更详细日志
