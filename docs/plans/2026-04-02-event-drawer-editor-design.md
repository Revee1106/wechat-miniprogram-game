# Event Drawer Editor Design

**Date:** 2026-04-02

**Goal**

将事件配置页从“左侧筛选 + 右侧整块表单”继续收缩成“左侧选择器 + 右侧摘要面板 + 右侧抽屉编辑”，进一步减少整页滚动。

## Problem

当前事件页已经移除了大部分冗余说明，但仍然存在两个问题：

- 左侧已经通过下拉框选择当前事件，底部再用事件标签云重复展示对象，信息冗余。
- 右侧虽然压缩成了固定高度工作区，但主体仍直接显示完整表单，字段较多时依然会在详情区产生较多滚动。

## Design Summary

事件页改为两层结构：

- 主工作区只展示当前事件的摘要和模块入口，不直接铺开表单。
- 点击模块入口后，从右侧滑出抽屉，在抽屉中完成模块级编辑。

保存逻辑保持不变，仍统一使用左侧保存按钮；抽屉只负责编辑，不负责提交。

## Layout

### Left Side

左侧只保留：

- `事件类型` 筛选
- `风险等级` 筛选
- `当前事件` 下拉
- `按类型新建` 下拉
- `保存事件` 按钮

移除事件标签云，不再重复展示当前筛选结果。

### Right Side

右侧主面板只保留三层信息：

- 当前事件标题
- 一行紧凑摘要标签
- 四个模块摘要卡

模块摘要卡为：

- `基础信息`
- `触发规则`
- `前置条件`
- `选项与结果`

每张卡片只展示摘要，不直接出现完整表单。

## Summary Content

### 基础信息

- 事件名称
- 事件编号
- 事件类型
- 结果倾向
- 标题文案
- 正文摘要首行

### 触发规则

- 风险等级
- 选项模式
- 触发来源
- 权重
- 境界范围
- 地域
- 冷却
- 单局上限
- 是否可重复

### 前置条件

- 资源条数
- 状态条数
- 功法条数
- 装备标签条数
- 最低转生
- 最低因果
- 最低气运

### 选项与结果

- 当前选项数
- 默认选项数
- 是否为单一结果
- 前两个选项名
- 超出部分显示“另 N 项”

## Drawer Interaction

- 点击任一模块卡，从右侧滑出抽屉。
- 抽屉标题即当前模块名。
- 抽屉内部显示该模块完整编辑表单。
- 抽屉支持内部滚动。
- 关闭抽屉后，主面板摘要立即反映最新编辑状态。

## Editing Rules

- `基础信息` 抽屉：展示名称、编号、类型、结果倾向、标题、正文。
- `触发规则` 抽屉：展示风险、触发来源、权重、地域、境界、冷却、上限、重复等字段。
- `前置条件` 抽屉：展示全部前置约束字段。
- `选项与结果` 抽屉：复用现有紧凑选项编辑器。

## Compatibility

- 不改后端 API。
- 不改事件保存、删除、运行时重载逻辑。
- 改动范围控制在 `admin-console/src/pages/EventListPage.tsx`、相关样式和测试。

## Validation

- 更新 Vitest 页面测试，验证：
  - 左侧不再渲染事件标签云
  - 点击模块卡可打开右侧抽屉
  - 抽屉内可见对应字段
- 最终执行：
  - `npm test -- --run`
  - `npm run build`
