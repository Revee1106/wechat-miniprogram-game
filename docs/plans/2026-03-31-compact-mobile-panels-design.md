# 洞府与炼丹页单屏化设计

日期：2026-03-31

## 背景

洞府页和炼丹页已经从事件页拆成独立页面，但当前布局仍然采用传统纵向堆叠：页头、总览、结果区、操作区、列表区依次向下排布。在手机视口里，这会导致用户进入页面后必须持续滚动，才能看到建造、炼丹、库存等关键操作。

用户希望进一步优化成“尽量不滚屏”的形式，并明确接受左右滑动或下拉等替代方案。

## 目标

1. 洞府页和炼丹页都尽量在单屏内完成主操作。
2. 顶部保留必要摘要和 `返回行程` 按钮。
3. 主要功能区改为横向切换，而不是整页纵向堆叠。
4. 需要浏览多个设施或多个丹方时，优先使用横向滑动卡片，而不是整页下拉。

## 方案比较

### 方案 A：只压缩字号和间距

优点：
- 改动小。

缺点：
- 只是“塞得更紧”，并没有改变信息密度。
- 洞府设施和丹方仍然会把页面拉长。

### 方案 B：顶部摘要 + 页签 + 横向滑动面板

优点：
- 更符合手机端单屏使用习惯。
- 把“总览”和“具体操作”分层。
- 洞府设施、丹方、库存都能用横向卡片展示。

缺点：
- 需要补一层页签状态和滑动联动。

### 方案 C：全部改成下拉折叠面板

优点：
- 信息仍在同一页。

缺点：
- 本质仍然依赖纵向滚动。
- 不符合这次“尽量不滚屏”的目标。

结论：采用方案 B。

## 交互设计

### 洞府页

- 顶部卷轴头部保留：
  - 标题
  - 一句说明
  - `返回行程`
  - 2 个关键指标：洞府等级、已建设施数
- 中部改成页签和 `swiper` 联动：
  - `总览`
  - `设施`
  - `月结`
- `总览`：展示当前洞府状态、一句营建建议、最近维护情况的摘要。
- `设施`：用横向卡片展示每个设施，每张卡片内保留建造或升级按钮。
- `月结`：只展示最近一次结算摘要和详细条目，避免主界面常驻大段列表。

### 炼丹页

- 顶部保留：
  - 标题
  - 一句说明
  - `返回行程`
  - 关键指标：炼丹术称号、药草、灵泉水
- 中部改成页签和 `swiper` 联动：
  - `丹方`
  - `炉次`
  - `库存`
- `丹方`：横向卡片展示丹方和开炉按钮。
- `炉次`：展示当前炉次和最近一次炼制结果。
- `库存`：横向卡片展示成丹库存与服用按钮。

## 技术实现

- 仅改前端小程序仓库。
- 每个页面新增本地状态：
  - `activePanel`
  - `currentPanelIndex`
- 使用：
  - 顶部分段按钮切换面板
  - `swiper` 左右滑动同步当前页签
- 洞府设施、丹方、库存列表改为 `scroll-view scroll-x` 的横向卡片带。
- 页面根容器改成固定视口高度的 `flex` 布局，内容区内部切换，不再整页无限向下增长。

## 风险与处理

- 风险：单张卡片内容过高，仍然挤爆视口。
  - 处理：将卡片文案压缩为摘要，不在卡片里堆完整说明。
- 风险：横向滑动与页签状态不同步。
  - 处理：统一由 `activePanel/currentPanelIndex` 管理，并在 `swiper` 的 `bindchange` 中同步。
- 风险：极小屏设备上按钮过密。
  - 处理：优先保证一屏一组主操作，次级信息放入其他页签，不强行同屏全展示。

## 验收标准

1. 洞府页进入后，无需长距离下拉就能看到顶部摘要和页签。
2. 洞府设施通过左右滑动卡片浏览和操作。
3. 炼丹页进入后，无需长距离下拉就能看到顶部摘要和页签。
4. 丹方和库存通过左右滑动卡片浏览与操作。
5. 两个页面仍保留 `返回行程` 按钮。
