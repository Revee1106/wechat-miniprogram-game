# 行程页布局重构设计

日期：2026-03-31

## 背景

当前行程页在无待处理事件时，采用“上方主卡片 + 下方 inspection-sheet 复合卡片”的结构。`inspection-sheet` 既承担底部标签切换，又承担标签内容展示，导致三个问题：

1. 标签导航和内容展示混在同一层，页面重点不清晰。
2. 顶部已经展示了“剩余寿元”，命牌区域仍再次展示寿元，信息重复。
3. 底部标签不是真正独立的导航区，视觉上更像第二张内容卡。

用户希望重新设计为：底部独立标签导航，屏幕其他区域作为展示区与功能区，并减少重复信息。

## 目标

1. 无待处理事件时，行程页重构为三段式：
   - 顶部：行程概览
   - 中部：主展示区 / 功能区
   - 底部：固定标签导航
2. `inspection-sheet` 只保留标签导航，不再承载内容展示。
3. `命牌` 标签仅展示必要人物信息，避免重复顶部已展示的寿元。
4. 有待处理事件时，继续保持事件优先模式，不重构事件决策视图。

## 方案比较

### 方案 A：保留现有上下双卡，只压缩信息

优点：
- 改动小。

缺点：
- 结构问题仍在，标签导航和内容仍混在一起。
- 重复信息只能靠删字段缓解，层级依然混乱。

### 方案 B：底部独立标签导航，中部单一主展示区

优点：
- 标签导航职责单一。
- 主展示区可按标签定制，不再受组件旧结构约束。
- 更接近移动端应用的“主内容 + 底部 tab”范式。

缺点：
- 需要重排 event 页结构，并调整 `inspection-sheet` 组件职责。

结论：采用方案 B。

## 交互设计

### 有待处理事件时

- 保持当前模式：
  - 事件卡片
  - 选项按钮
- 不引入底部固定 tab 结构，避免在决策状态下分散注意力。

### 无待处理事件时

- 顶部 `journey-overview`：
  - 已行时间
  - 剩余寿元
  - 一句当前状态摘要
  - 如有最近事件结果，展示压缩版列表
- 中部 `journey-main-stage`：
  - 依据当前底部标签切换主内容
- 底部 `journey-dock`：
  - `命牌`
  - `行囊`
  - `修炼`
  - `洞府`

## 各标签内容

### 命牌

- 展示：
  - 当前境界
  - 修为进度
  - 气运
- 不再展示寿元。

### 行囊

- 展示：
  - 灵石
  - 药草
  - 玄铁精华
  - 其他关键资源摘要

### 修炼

- 展示：
  - 当前状态
  - 突破提示
  - 门槛
  - 破境按钮

### 洞府

- 展示：
  - 洞府等级
  - 已建设施数
  - 最近月结一句摘要
  - `洞府营建`
  - `炼丹房`

## 技术实现

- `pages/event/event.wxml`
  - 重构无事件状态布局
  - 中部主展示区直接按 `activeSection` 分支渲染内容
- `pages/event/event.wxss`
  - 新增顶部概览条、主展示区、底部 dock 的样式
- `pages/event/event.js`
  - 继续复用现有运行态数据，不增加复杂状态
- `components/inspection-sheet`
  - 收缩为“纯标签导航”组件
  - 移除原本的信息展示 body

## 风险与处理

- 风险：原 `inspection-sheet` 的内容移走后，event 页模板会显著膨胀。
  - 处理：只搬运行程页自己的核心展示，不把子页面逻辑也挪进来。
- 风险：样式重构可能影响小屏幕下按钮布局。
  - 处理：中部主展示区始终只显示一个标签内容，底部 dock 独立固定，减小拥挤度。
- 风险：事件模式与普通行程模式视觉差异过大。
  - 处理：继续沿用同一套纸卷色系和主按钮语言，保证风格统一。

## 验收标准

1. 无待处理事件时，底部是独立的标签导航区域。
2. `inspection-sheet` 不再展示人物/资源/修炼/洞府正文。
3. 命牌标签里不再显示寿元字段。
4. 顶部仍可看到剩余寿元。
5. 洞府和修炼标签仍保留现有主功能入口。
