# 微信小游戏后端已完成功能说明

最后更新：2026-04-02

本文档仅记录 `wechat-miniprogram-game` 仓库内已经落地、且可从当前代码与测试中确认的能力，不包含仅存在于设计稿或计划文档中的内容。

## 一、核心主循环

- 已提供健康检查接口 `GET /api/health`。
- 已提供单局创建、状态读取、时间推进、事件结算、突破、转生等主循环接口。
- 运行态已覆盖角色寿元、修为、境界、死亡状态、当前事件、资源、洞府、炼丹与总结文案等核心字段。
- 主循环服务已统一处理常见业务错误并映射为 HTTP 400 / 404 / 409。

## 二、事件系统

- 事件模板与事件选项已配置化，当前运行依赖 `config/events/templates.json` 与 `config/events/options.json`。
- 已支持按条件过滤事件，包括境界与运行态相关限制。
- 已支持“事件类型权重 + 事件权重”两层随机抽取。
- 已支持单结果事件与多选项事件。
- 已支持结构化结果 payload 结算，事件结果可直接写入角色数值与资源。
- 已支持运行时重载事件配置，后台保存后可刷新到运行时。

## 三、境界与突破

- 境界配置已从种子数据抽离到 `config/realms/realms.json`。
- 已提供完整的境界列表读取、单条读取、创建、更新、删除、排序、校验、重载接口。
- 突破逻辑已读取实时境界配置，而不是写死在代码中。
- 事件过滤与突破门槛已接入当前境界顺序。

## 四、事件/境界/洞府配置后台

- FastAPI 已挂载后台接口与静态管理台入口。
- 已提供后台登录、登出、会话查询接口。
- 事件配置后台已支持事件列表、详情、创建、编辑、删除、选项增删改、校验、运行时重载。
- 境界配置后台已支持境界列表、详情、创建、编辑、删除、重排、校验、运行时重载。
- 洞府配置后台已支持设施列表、详情、按等级编辑、追加新等级、校验、运行时重载。
- 后台前端已具备事件、境界、洞府三套编辑视图，且带登录态保护。
- 后台前端三套配置页已统一为紧凑工作台样式：左侧工具栏负责选择与保存，右侧通过小标签和抽屉完成编辑，减少整页滚动。
- 事件配置页已支持右侧抽屉编辑基础信息、触发规则、前置条件、选项与结果，并统一保存后自动重载运行时。
- 事件前置条件与选项前置条件已支持“空状态仅展示新增按钮”，新增条件时优先通过下拉选择条件类型。
- 事件结果编辑器已统一为“变化项列表 + 附加变化”模式，资源与修为等数值变化共用一套编辑入口，未配置的变化项默认不展示。

## 五、经济与资源 V1

- 已引入 `app/economy` 领域层，并提供资源目录与轮回点结算配置。
- 运行态已支持统一资源栈 `resource_stacks`，角色档案已支持 `rebirth_points`。
- 已支持资源出售接口 `POST /api/run/resource/sell`。
- 已支持灵石转化修为接口 `POST /api/run/resource/convert-cultivation`，当前规则为消耗灵石换取修为。
- 已支持转生时按境界、存活轮数、稀有资源等维度结算轮回点。

## 六、洞府系统 V1

- 运行态已支持洞府等级、设施状态与最近一次月结结果。
- 洞府规格已改为从 `config/dwelling/facilities.json` 读取，游戏启动默认加载该配置。
- 已落地 5 个洞府设施：
  - 灵田
  - 灵泉
  - 矿洞
  - 炼丹房
  - 聚灵阵
- 已支持设施建造与升级。
- 已支持时间推进时的月度维护扣费、资源产出与停摆结算。
- 已支持维护不足导致设施停摆。
- 聚灵阵已接入月度修为收益与突破辅助加成。
- 炼丹房已作为炼丹玩法的前置解锁条件之一。
- 已提供洞府建造与升级接口：
  - `POST /api/run/dwelling/build`
  - `POST /api/run/dwelling/upgrade`

## 七、炼丹系统 V1

- 运行态已支持 `alchemy_state`，包含可用丹方、当前炼制任务、库存、熟练度等级等信息。
- 已落地 6 个基础丹方：
  - 辟谷丹
  - 养气丹
  - 养元丹
  - 宁神丹
  - 聚灵丹
  - 固元丹
- 已支持开炉炼制，并可选择是否消耗灵泉水辅助炼制。
- 已支持时间推进后结算炼丹结果。
- 已支持成丹 / 废丹结果与下品 / 中品 / 上品品质。
- 已支持丹药库存累积与服用消耗。
- 已提供炼丹接口：
  - `POST /api/run/alchemy/start`
  - `POST /api/run/alchemy/consume`

## 八、当前仍未覆盖为“完整玩法”的部分

- 战斗系统尚未落地。
- 更完整的资源生产、交易、价格波动与订单体系尚未落地。
- 洞府仍是 V1 经营形态，尚未形成更复杂的布局、相邻加成、排班等系统。
- 炼丹已具备最小闭环，但尚未扩展到更高阶炉火、批量排产或复杂失败惩罚。
