# 问道长生后端与配置后台进度说明

本仓库当前承载的是《问道长生》微信小游戏的后端主循环、事件与境界配置后台，以及一部分经济系统 V1 基础能力。

截至 `2026-03-27`，仓库状态更接近“核心纵切版本已跑通”，不是“完整玩法版图已完成”。

## 当前已完成

### 1. 核心主循环

- 可创建单局运行
- 可推进时间
- 可触发并结算事件
- 可执行突破
- 可处理死亡
- 可执行转生并开启下一局

### 2. 事件系统

- 事件模板与选项已配置化
- 支持条件过滤
- 支持事件类型权重 + 事件权重两层随机
- 支持单结果事件与多选项事件
- 支持结构化结果 payload 结算
- 支持运行时重载事件配置

### 3. 事件配置后台

- 已有登录保护
- 已支持事件增删改查
- 已支持结果配置编辑
- 已支持中文界面
- 已支持权重展示
- 已支持保存后自动重载运行时

### 4. 境界与突破系统

- 境界配置已从 seed 抽离到 `config/realms/realms.json`
- 已建立四段小层级境界链路
- 突破读取实时境界配置
- 事件境界筛选读取实时境界顺序

### 5. 境界配置后台

- 已支持境界增删改查
- 已支持境界排序
- 已支持删除/禁用时的事件引用保护
- 已支持保存和排序后实时重载

### 6. 经济系统 V1 已落地部分

- 已新增 `app/economy` 领域层
- 已新增资源目录配置 `config/economy/resources.json`
- 已新增转生结算权重配置 `config/economy/settlement.json`
- `RunState` 已支持 `resource_stacks`
- `PlayerProfile` 已支持 `rebirth_points`
- 转生时会结算轮回点
- 事件奖励已可写入统一资源栈
- 突破配置已预留 `required_materials` 结构，为后续材料型突破做兼容

当前资源集为：

- `spirit_stone`
- `basic_herb`
- `basic_ore`
- `basic_breakthrough_material`
- `rare_material`

当前轮回点结算维度为：

- 境界顺序权重
- 存活回合数
- 稀有资源数量
- 特殊事件数接口预留

## 当前部分完成

### 1. 经济系统

- 已有基础资源定义、资源栈与轮回点结算
- 还没有形成完整的生产 / 消耗 / 转化闭环
- 事件需求仍以旧资源字段兼容为主
- 突破还不会真正消耗 `required_materials`
- 特殊事件数还没有接入专门的运行时统计
- 经济配置仍是本地 JSON，尚未进入独立后台工作流

### 2. 洞府系统

- 已有基础加成与入口
- 还不是完整经营玩法

### 3. 前端页面体系

- 主路径页面结构已经搭起来
- 但并非每个页面都具备完整玩法深度

## 当前未完成或未展开

- 战斗系统
- 炼丹 / 炼器 / 制作完整玩法
- 更完整的资源掉落与循环体系
- 广告激励
- 更通用的配置后台平台化能力

## 当前可用程度

如果以“可持续开发与可配置维护”为标准，本仓库已经具备：

- 可运行的后端核心循环
- 可维护的事件配置体系
- 可维护的境界配置体系
- 经济系统 V1 的最小资源与结算骨架

如果以“完整游戏玩法版图”为标准，当前仍处于中前期。
